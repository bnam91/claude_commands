상품 상세페이지 URL을 받아서 판매자가 내세우는 구매 이유(USP)와 페이지 구성 전략을 분석하고 맥 노트에 저장하는 스킬이야.

## 실행 순서

### 1단계: 브라우저로 페이지 열기

```
mcp__chrome-devtools__new_page(url="{입력받은 URL}")
```

### 2단계: 캡차/보안 확인 처리

- `take_screenshot`으로 현재 상태 확인
- 네이버 캡차 등 보안 확인 화면이 뜨면:
  - `take_snapshot`으로 질문 텍스트와 이미지 확인
  - 이미지 속 정보를 읽어 정답 추론
  - `fill(uid, value)`로 정답 입력
  - `click` 으로 확인 버튼 클릭
  - 로그인 요구 dialog가 뜨면 `handle_dialog(action="dismiss")`로 닫기

### 3단계: 상세정보 펼치기

스크롤 전에 반드시 "상세정보 펼쳐보기" 버튼 클릭:

```
evaluate_script: () => {
  const btns = document.querySelectorAll('button, div, span, a');
  for(const btn of btns) {
    if(btn.innerText && btn.innerText.includes('상세정보 펼쳐보기')) {
      btn.click(); return '클릭됨';
    }
  }
  return '버튼없음';
}
```

버튼이 보이지 않으면 먼저 스크롤해서 버튼 위치까지 내려간 후 클릭. 클릭 후 펼쳐진 상태에서 해당 위치부터 다시 스크롤.

### 4단계: 상세페이지 스크롤 & 읽기

아래 루프를 페이지 끝까지 반복:

```
evaluate_script("() => { window.scrollBy(0, 2000); return window.scrollY; }")
take_screenshot()
```

**읽으면서 수집할 정보:**
- 상품명, 가격, 할인율
- 판매자가 강조하는 핵심 메시지 (이미지 텍스트 포함)
- 배송 관련 어필 (속도, 당일발송 등)
- 고객 서비스 관련 어필 (상담, 환불 정책 등)
- 신뢰 지표 (리뷰 수, 평점, 재구매율 등)
- 제품 차별점 (소재, 품질, 호환성 등)
- 가격 경쟁력 어필

페이지가 리뷰 섹션에 진입하면 상세페이지 읽기 완료로 판단.

### 5단계: 분석 정리

수집한 내용을 아래 두 가지로 정리:

**① USP 분석**
```
【상품명】 USP 분석
분석일: {오늘 날짜}
URL: {입력 URL}

■ 핵심 USP 한 줄 요약

■ USP 목록
1. [카테고리] 내용
2. [카테고리] 내용
...

■ 신뢰 지표
- 리뷰 수, 평점, 재구매율 등

■ 가격 정보
- 정가 / 판매가 / 할인율
```

**② 판매 페이지 구성**
```
■ 섹션별 구성 순서
1. {섹션명}: {목적}
2. {섹션명}: {목적}
...

■ 전체 설득 흐름
문제 공감 → ... → 구매 유도
```

### 5단계: 맥 노트 저장

아래 명령으로 "USP 분석" 폴더에 저장:

```bash
python3 ~/Documents/claude_skills/app_notes_control/app_notes_control.py \
  --create \
  --title "{상품명} USP 분석 ({오늘날짜})" \
  --body "{정리한 내용}" \
  --folder "USP 분석"
```

폴더가 없으면 자동 생성되니 그냥 실행해.

### 6단계: 결과 보고

사용자에게 분석 결과를 마크다운으로 출력하고,
"맥 노트 'USP 분석' 폴더에 저장했어요." 메시지로 마무리.

## 주의사항

- 스크린샷은 이미지 텍스트 읽기용으로만 사용 (토큰 소비 최소화)
- 리뷰 섹션 진입 시 스크롤 중단
- 캡차가 반복되거나 로그인 필수인 경우 사용자에게 안내
- 노트 저장 시 body 인자는 반드시 따옴표로 감싸고 특수문자 이스케이프 처리
