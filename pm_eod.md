PM 마감 브리핑 스킬이야. 콜아웃 + 프로젝트 DB를 읽어서 PM 분석이 담긴 업무 마감 브리핑을 브리핑 DB에 기록해.

## PM 책임

- **단순 완료율 집계 금지** — "완료 3/4" 수치만 쓰는 건 PM 역할이 아님
- 미완료 항목은 반드시 **왜 안 됐는지, 어떤 조건에 걸려있는지** 분석해서 코멘트 작성
- 조건부 대기 항목은 **언제 해제되는지, 해제 안 되면 런칭에 어떤 영향인지** 명시
- 내일 우선순위는 **D-day 임박도 + 블로킹 여부 + 의존성** 기준으로 순번 결정
- briefing_rules.md 문체 규칙 준수: 서술형 종결 (~합니다, ~됩니다), 축약형/명사형 종결 금지

## 실행 순서

### 1단계 — 데이터 수집

```bash
cd ~/Documents/claude_skills/notion && node _eod_write.js --pm gg --json
# 날짜 지정 시: node _eod_write.js --pm gg --json --date 3.9
# CC 프로젝트: node _eod_write.js --pm cc --json
```

→ 콜아웃(완료/미완료 task) + 프로젝트 DB(블로킹/진행중/대기 항목) + 지혜 업무코멘트 출력

### 2단계 — Claude PM 분석 및 브리핑 초안 작성

1단계 JSON을 바탕으로 Claude가 아래 구조로 브리핑을 분석·작성:

```
현빈02 — 완료 N/M
  ✅ 완료 task (코멘트 없음)
  ⬜ 미완료 task → 미완료 이유 + 내일 연동 방식 서술

지혜 — 완료 N/M
  (동일 구조)

지혜 업무코멘트 주요 내용
  핵심 내용 bullet

⏳ 조건부 대기
  항목 → PM 판단 (해제 조건, D-day 리스크)

PM 총평 — 내일(M.D) 우선순위
  1. [담당] 업무명 → 판단 이유
  2. ...
```

### 3단계 — 브리핑 DB 기록

Claude가 분석한 내용을 아래 JSON 스키마로 구성해서 스크립트에 넘김:

```json
{
  "members": [
    {
      "name": "현빈02",
      "done": ["task1", "task2"],
      "undone": [
        { "task": "task명", "comment": "미완료 이유 및 내일 연동 서술" }
      ]
    }
  ],
  "comments": ["지혜 코멘트 요약1", "요약2"],
  "conditionalWaiting": [
    "항목명 → PM 판단 (조건, 리스크 서술)"
  ],
  "pmSummary": [
    "[담당] 업무명 → 판단 이유 (D-day 영향 포함)"
  ]
}
```

```bash
cd ~/Documents/claude_skills/notion && node _eod_write.js --pm gg --write '<위 JSON>'
```

## 브리핑 품질 기준

❌ 나쁜 예 (집계만):
```
📊 전체 완료율 5/7
✅ 현빈02 3/4 — 상세페이지 기획, 딜롱 단가협의 완료
⬜ 현빈02 미완료 — 본품 퀄리티 체크
```

✅ 좋은 예 (PM 판단 포함):
```
현빈02 — 완료 3/4
  ✅ 🚨 상세페이지 기획 v.0.5
  ✅ 딜롱 주문할 옵션 선정 및 단가 협의
  ⬜ 본품 도착 및 퀄리티 체크 (ID라벨링 준비)
    딜롱 샘플이 3.9 도착했으나 검수 미완료 상태입니다.
    3.10 검수 후 본품 주문 GO/NO-GO 결정과 함께 처리가 필요합니다.

⏳ 조건부 대기
  본품 주문 1차 (데드라인 3.12) → 딜롱 샘플 도착 확인됨.
  3.10 검수 결과 기반으로 GO 결정 가능합니다.
  결정 지연 시 3.12 데드라인 미스 위험이 있습니다.

PM 총평 — 내일(3.10) 우선순위
  1. [현빈] 본품 주문 1차 GO/NO-GO → 데드라인 3.12, 오늘 결정하지 않으면
     내일부터 일정 압박이 시작됩니다.
  2. [현빈] 크몽 디자이너 섭외 → 상페 제작 블로킹의 유일한 해소 경로입니다.
```
