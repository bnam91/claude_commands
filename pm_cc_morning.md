CC DB와 주간회의를 자동 조회해서 수지(진행중+우선순위1)/현빈(업무막힘) 기준으로 업무를 배정하고 morning meeting 토글에 브리핑을 입력하는 스킬이야.

브리핑 초안 작성 → 현빈 검토 → 실제 콜아웃 전송 3단계로 실행돼.

## PM 책임

- **이 스킬의 핵심 목적은 주간 목표를 완수시키는 것** — 매일 오전 브리핑이 그 수단이다
- 오늘 배정 업무가 이번주 목표와 연결되어 있는지 2단계에서 반드시 확인
- 브리핑 초안은 자동 생성이지만 **내용의 정확성과 업무 배정 적절성은 현빈이 책임진다**
- 2단계 검토 없이 3단계를 실행하면 안 됨 — 잘못된 업무가 실제 콜아웃에 배정됨
- 막힘 항목이 있으면 브리핑에서 확인 후 **오늘 중 해소 방향을 결정하고** 3단계 실행
- morning meeting content는 팀원이 읽는 글 — 구어체로 맥락이 담겨야 함

## 배정 기준

- **수지**: 현재 진행 중 우선순위1~2 + 진행대기 상위5 (1~3단계 실행 업무)
- **현빈**: 업무막힘 항목 (GO/NO-GO 결정, 외부 연락 등 현빈만 처리 가능한 것)

> 현빈은 시스템 설계·방향 결정만. 실행 업무는 수지에게.

## 실행 순서

### 1단계 — 브리핑 DB 초안 작성
CC DB + 주간회의 pm-cc 이번주 목표를 읽어서 브리핑 초안 작성 (실제 콜아웃 전송 X)

```bash
cd ~/Documents/claude_skills/notion && node _pm_briefing.js --pm cc --json
```

→ 데이터 출력 후 Claude가 주간목표 + HR.md 기반으로 수지/현빈 배분 결정
→ 결정된 내용으로 브리핑 DB 초안 작성

```bash
cd ~/Documents/claude_skills/notion && node _pm_briefing.js --pm cc
```

### 2단계 — 브리핑 검토
현빈이 브리핑 DB 오늘 페이지 확인 후 수정 (필요 시)

### 3단계 — 실제 콜아웃 전송 + morning meeting 입력
현빈이 2단계에서 수정한 브리핑 초안을 기준으로 실제 콜아웃에 업무 배정 + morning meeting 토글 입력

먼저 전송 내용 JSON으로 확인 (선택):
```bash

```

실제 전송:
```bash

```

```bash
node ~/Documents/claude_skills/agent_meeting/agent_pm_input.js --pm pm-cc --type morning --content "..."
```

#### morning meeting content 작성 규칙 (구어체 서술형, 2~4줄)
- 불릿/통계 나열 금지 — 흐름 있는 문장으로 작성
- 주간회의(weekly meeting) 내용 먼저 읽고 이번주 컨텍스트 반영
- D-day 언급 + 오늘 핵심 상황 한 줄
- 막힘·선행 조건 있으면 "그게 안 되면 ~가 막혀" 식으로 맥락 포함
- 배정 건수보다 오늘의 흐름과 우선순위를 말로 설명
- PM이 팀원에게 직접 말하듯 작성

## 노션 업무요청 입력 규칙

### 날짜 토글 생성
- 콜아웃 내 `M.D` 형식 토글로 생성 (예: `3.9`, `2.27`)
- 해당 날짜 토글이 이미 있으면 새로 만들지 않고 기존 토글에 추가
- 없을 때만 토글 + 하위 체크박스를 한 번에 생성

### 업무 추가
- 날짜 토글 하위에 템플릿 형식으로 추가, 미완료 상태
- 인풋 / 아웃풋 / 📍 맥락 포함 (내용은 현빈이 직접 입력)

### 앵커블록 (삭제/이동 금지)
- `업무요청_` 텍스트 포함 paragraph → 항상 유지
- `📌`로 시작하는 토글 → 항상 유지
- 7일 이내 날짜 토글/heading → 유지
- 맨 끝 빈 paragraph → 유지
- 7일 이상 지난 날짜 토글 → 이전 페이지로 이동 (삭제 아님)
